# 部署管理用户文档

## 概览

部署管理系统是 OpsPilot 平台的核心功能，提供从基础设施管理到应用发布的完整生命周期管理能力。

### 核心功能

- **基础设施管理**: 凭证、集群、主机管理
- **部署目标管理**: 创建和配置部署环境
- **发布管理**: 创建、审批、部署、回滚
- **可观测性**: 拓扑、指标、审计、策略、AIOps

---

## 快速开始

### 1. 准备基础设施

#### 1.1 注册凭证
1. 访问 **基础设施 > 凭证管理**
2. 点击"注册凭证"
3. 选择运行时类型（Kubernetes/Docker）
4. 填写凭证信息
5. 测试连接

#### 1.2 创建集群（可选）
1. 访问 **基础设施 > 集群管理**
2. 点击"引导集群"
3. 按向导步骤配置：
   - 基本信息（名称、版本）
   - 控制平面节点
   - 工作节点
   - CNI 插件
4. 提交并等待引导完成

### 2. 创建部署目标

1. 访问 **部署目标 > 创建目标**
2. 填写基本信息：
   - 名称
   - 环境（开发/预发布/生产）
   - 描述
3. 选择运行时和凭证
4. 绑定主机或节点
5. 配置引导选项
6. 提交创建

### 3. 引导环境

1. 在目标详情页点击"引导环境"
2. 选择运行时包：
   - Istio（服务网格）
   - Prometheus（监控）
   - Grafana（可视化）
   - 其他组件
3. 配置包参数
4. 启动引导
5. 观察实时进度

### 4. 创建发布

1. 访问 **发布管理 > 创建发布**
2. 步骤 1: 选择服务和目标
3. 步骤 2: 配置变量
4. 步骤 3: 预览清单
5. 步骤 4: 选择部署策略
6. 步骤 5: 确认提交

### 5. 审批发布（生产环境）

1. 访问 **发布管理 > 审批中心**
2. 查看待审批列表
3. 点击发布查看详情
4. 审批或拒绝：
   - 审批：添加意见并确认
   - 拒绝：说明原因并确认

## 功能详解

### 凭证管理

#### 注册平台凭证
适用于平台管理的集群，自动关联凭证信息。

**步骤**:
1. 点击"注册凭证"
2. 填写表单：
   - 凭证名称
   - 运行时类型
   - 关联集群
   - 描述
3. 提交

#### 导入外部凭证
适用于外部集群，手动导入 kubeconfig 或证书。

**步骤**:
1. 点击"导入凭证"
2. 选择导入方式：
   - 上传 kubeconfig 文件
   - 粘贴证书内容
3. 填写凭证名称
4. 提交

#### 测试凭证
验证凭证是否可用。

**步骤**:
1. 在凭证列表中找到目标凭证
2. 点击"测试连接"
3. 查看测试结果：
   - 成功：显示延迟
   - 失败：显示错误信息

---

### 集群管理

#### 集群引导
创建新的 Kubernetes 集群。

**步骤**:
1. 访问 **集群管理 > 引导集群**
2. 步骤 1 - 基本信息：
   - 集群名称
   - Kubernetes 版本
   - 描述
3. 步骤 2 - 控制平面：
   - 选择主机
   - 配置高可用（可选）
4. 步骤 3 - 工作节点：
   - 选择主机
   - 配置节点标签
5. 步骤 4 - CNI 插件：
   - Calico
   - Flannel
   - Cilium
6. 预览配置
7. 提交引导

#### 引导进度跟踪
实时查看集群引导进度。

**显示内容**:
- 阶段指示器（预检、安装、验证）
- 实时日志流
- 进度百分比
- 错误信息（如果失败）
- 回滚信息（如果失败）

---

### 部署目标管理

#### 创建目标
定义应用部署的目标环境。

**步骤**:
1. 访问 **部署目标 > 创建目标**
2. 步骤 1 - 基本信息：
   - 目标名称
   - 环境类型（dev/staging/production）
   - 描述
3. 步骤 2 - 运行时配置：
   - 运行时类型（Kubernetes/Docker）
   - 选择凭证
4. 步骤 3 - 绑定资源：
   - Kubernetes: 选择命名空间
   - Docker: 选择主机
5. 步骤 4 - 引导选项：
   - 是否立即引导
   - 选择运行时包
6. 提交创建

#### 环境引导
为目标环境安装运行时组件。

**可用组件**:
- **Istio**: 服务网格，流量管理
- **Prometheus**: 指标收集和监控
- **Grafana**: 可视化仪表板
- **Jaeger**: 分布式追踪
- **Fluentd**: 日志收集

**步骤**:
1. 在目标详情页点击"引导环境"
2. 选择需要的组件
3. 配置组件参数
4. 启动引导
5. 观察进度：
   - 预检：验证环境
   - 安装：部署组件
   - 验证：健康检查

---

### 发布管理

#### 创建发布
部署应用到目标环境。

**步骤**:
1. 访问 **发布管理 > 创建发布**
2. 步骤 1 - 选择服务和目标：
   - 搜索并选择服务
   - 选择部署目标
   - 选择镜像版本
3. 步骤 2 - 配置变量：
   - 系统自动检测模板变量
   - 填写变量值
   - 支持环境变量、配置文件
4. 步骤 3 - 预览清单：
   - 查看生成的 YAML/Dockerfile
   - 查看检查结果和警告
   - 确认配置正确
5. 步骤 4 - 部署策略：
   - **滚动更新**: 逐步替换实例
   - **蓝绿部署**: 新旧版本切换
   - **金丝雀发布**: 灰度发布
6. 步骤 5 - 确认：
   - 查看汇总信息
   - 生产环境会显示审批提示
   - 提交发布

#### 部署策略说明

**滚动更新 (Rolling Update)**
- 逐步替换旧版本实例
- 零停机时间
- 适合大多数场景
- 配置项：
  - 最大不可用数
  - 最大超出数

**蓝绿部署 (Blue-Green)**
- 部署新版本（绿）
- 测试通过后切换流量
- 可快速回滚
- 需要双倍资源
- 配置项：
  - 测试时长
  - 自动切换

**金丝雀发布 (Canary)**
- 先发布到小部分实例
- 逐步增加流量比例
- 降低风险
- 配置项：
  - 初始流量比例
  - 增量步长
  - 观察时长

#### 审批工作流
生产环境发布需要审批。

**审批流程**:
1. 创建生产环境发布
2. 系统自动创建审批票据
3. 审批人收到通知
4. 审批人访问 **审批中心**
5. 查看发布详情：
   - 服务信息
   - 目标环境
   - 变更内容
   - 风险评估
6. 做出决定：
   - **审批**: 添加意见，发布自动执行
   - **拒绝**: 说明原因，发布被取消

**审批权限**:
- 需要 `deployment:approve` 权限
- 可在 **访问治理 > 角色管理** 中配置

#### 实时进度跟踪
查看发布执行进度。

**显示内容**:
- **状态流**: 可视化状态转换
- **进度条**: Pod 部署进度（Kubernetes）
- **健康检查**: 探针结果
- **实时日志**: 部署日志流
- **时间线**: 关键事件记录

**自动刷新**:
- 每 10 秒自动刷新
- 到达终态后停止刷新
- 终态：applied（成功）、failed（失败）、rejected（拒绝）

#### 回滚
将应用回滚到之前的版本。

**步骤**:
1. 访问成功的发布详情页
2. 点击"回滚"按钮
3. 确认回滚操作
4. 系统创建回滚发布
5. 观察回滚进度

**注意事项**:
- 只能回滚成功的发布
- 回滚会创建新的发布记录
- 回滚发布显示源发布 ID
- 生产环境回滚也需要审批

### 可观测性

#### 部署拓扑
可视化服务部署关系。

**功能**:
- 服务节点显示部署状态
- 环境分组
- 交互式节点选择
- 实时状态更新

**状态颜色**:
- 🟢 绿色：健康
- 🟡 黄色：降级
- 🔴 红色：故障
- ⚪ 灰色：未知

#### 指标仪表板
查看部署相关指标。

**关键指标**:
- **部署频率**: 每日/周/月部署次数
- **成功率**: 部署成功百分比
- **平均部署时长**: MTTR
- **审批响应时间**: 审批平均耗时

**环境对比**:
- 开发环境
- 预发布环境
- 生产环境

#### 审计日志
记录所有操作历史。

**功能**:
- 操作类型过滤
- 操作人过滤
- 日期范围过滤
- 全文搜索
- 导出 CSV/JSON

**记录内容**:
- 操作时间
- 操作人
- 操作类型
- 操作对象
- 详细信息

#### 策略管理
配置服务治理策略。

**策略类型**:
- **流量策略**: 限流、熔断、重试
- **弹性策略**: 超时、重试、降级
- **访问策略**: 认证、授权、黑白名单
- **SLO 策略**: 可用性、延迟、错误率目标

**配置方式**:
1. 选择服务
2. 选择策略类型
3. 填写策略配置（JSON）
4. 验证并保存

#### AIOps 洞察
AI 驱动的智能分析。

**功能**:
- **风险发现**: 配置、性能、安全风险
- **异常检测**: 指标异常、趋势异常
- **优化建议**: 资源优化、部署优化、成本优化

**风险级别**:
- 🔴 高风险：需要立即处理
- 🟡 中风险：建议尽快处理
- ⚪ 低风险：可以延后处理

## 最佳实践

### 环境管理
1. **环境隔离**: 开发、预发布、生产严格隔离
2. **命名规范**: 使用统一的命名规范
3. **标签管理**: 为资源添加有意义的标签

### 发布管理
1. **小步快跑**: 频繁小规模发布，降低风险
2. **先测试后发布**: 在开发/预发布环境充分测试
3. **灰度发布**: 生产环境使用金丝雀策略
4. **监控告警**: 发布后密切监控指标

### 审批流程
1. **明确审批人**: 为每个环境指定审批人
2. **审批时限**: 设置合理的审批时限
3. **审批意见**: 记录审批决策依据

### 回滚策略
1. **快速回滚**: 发现问题立即回滚
2. **保留历史**: 保留足够的历史版本
3. **回滚测试**: 定期测试回滚流程

## 常见问题

### Q: 凭证测试失败怎么办？
A: 检查以下几点：
1. 凭证信息是否正确
2. 网络连接是否正常
3. 目标集群是否可访问
4. 凭证权限是否足够

### Q: 集群引导失败如何处理？
A: 查看引导日志，常见原因：
1. 主机资源不足
2. 网络配置问题
3. 镜像拉取失败
4. 配置参数错误

### Q: 发布卡在 pending 状态？
A: 可能原因：
1. 等待审批（生产环境）
2. 资源不足（Pod 无法调度）
3. 镜像拉取失败
4. 健康检查失败

### Q: 如何加快部署速度？
A: 优化建议：
1. 使用本地镜像仓库
2. 优化镜像大小
3. 调整部署策略参数
4. 增加资源配额

### Q: 审批流程可以自定义吗？
A: 当前版本支持：
1. 配置审批人
2. 设置审批权限
3. 自定义审批规则（通过策略）

## 故障排查

### 部署失败
1. 查看发布详情页的错误信息
2. 查看实时日志
3. 检查目标环境状态
4. 验证配置和变量

### 回滚失败
1. 确认源发布状态正常
2. 检查目标环境资源
3. 查看回滚日志
4. 必要时手动干预

### 审批通知未收到
1. 检查用户权限
2. 确认通知配置
3. 查看审批中心

### 实时日志不更新
1. 检查网络连接
2. 刷新页面
3. 查看浏览器控制台错误

## API 参考

详细 API 文档请参考：`/api/docs/deployment`

### 主要 API 端点

**凭证管理**:
- `GET /api/v1/deployment/credentials` - 获取凭证列表
- `POST /api/v1/deployment/credentials` - 创建凭证
- `POST /api/v1/deployment/credentials/:id/test` - 测试凭证

**集群管理**:
- `GET /api/v1/deployment/clusters` - 获取集群列表
- `POST /api/v1/deployment/clusters/bootstrap` - 引导集群
- `GET /api/v1/deployment/clusters/:id` - 获取集群详情

**部署目标**:
- `GET /api/v1/deployment/targets` - 获取目标列表
- `POST /api/v1/deployment/targets` - 创建目标
- `POST /api/v1/deployment/targets/:id/bootstrap` - 引导环境

**发布管理**:
- `GET /api/v1/deployment/releases` - 获取发布列表
- `POST /api/v1/deployment/releases/preview` - 预览发布
- `POST /api/v1/deployment/releases/apply` - 创建发布
- `POST /api/v1/deployment/releases/:id/approve` - 审批发布
- `POST /api/v1/deployment/releases/:id/rollback` - 回滚发布
