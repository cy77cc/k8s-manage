# Ops FAQ 100 知识喂料说明

## 数据源

- 用户可读版本：`docs/user/ops-faq-100.md`
- 帮助中心主手册：`docs/user/help-center-manual.md`
- 一题一条 JSONL：`docs/ai/ops-faq-100.jsonl`
- 后端加载入口：`internal/service/ai/faq_knowledge.go`

## 推荐喂料策略

1. 将 FAQ 文档按每 5-10 题切分为片段。
2. 为每个片段生成向量并建立检索索引。
3. 用户提问时先检索最相关 2-4 个片段。
4. 将检索片段 + 实时上下文（告警/主机/服务）一起注入提示词。

## 回答约束

- 涉及实时状态必须提示“需实时查询确认”。
- 涉及变更操作必须提示审批与回滚方案。
- 不确定时先问关键补充信息，不要猜。

## 建议系统提示词

你是企业 DevOps 运维平台助手。
请优先给出可执行步骤，并遵循：
- 结构：路径 -> 步骤 -> 风险 -> 回滚/下一步
- 实时状态需声明需要实时数据确认
- 变更动作必须提醒审批约束
- 回答控制在必要信息，不输出无关内容
