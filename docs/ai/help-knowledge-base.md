# AI 帮助知识库（喂给 AI）

## 用途

该文档用于给平台 AI 助手提供产品使用知识（功能入口、标准流程、常见故障、操作边界）。

建议优先用于：
- FAQ 问答
- 运营值班支持
- 操作路径引导
- 变更前风险提示

## 知识边界

- 若涉及实时状态（当前告警、主机在线性、发布结果），AI 必须结合实时接口数据，不得仅依据本知识库下结论。
- 若涉及变更执行，必须遵守审批策略与权限校验。

## 结构化知识

### K1 快速上手
- 登录后先切换项目，再看监控健康度与 critical 告警。
- 日常先看监控，再看主机与服务。

### K2 主机纳管
- 路径：主机管理 -> 主机纳管。
- 关键步骤：填连接信息 -> 连通性探测 -> 提交纳管 -> 在线确认。
- 常见失败：网络不通、凭据错误、账号权限不足。

### K3 告警处置
- 先按严重级别排序（critical 优先）。
- 检查触发指标、对象、时间窗。
- 联动 AI 做只读诊断（CPU/内存/磁盘/日志）。

### K4 服务发布
- 标准流程：预览 -> 风险确认 -> 执行 -> 观测 -> 回滚（必要时）。
- 生产环境高风险变更必须二次确认。

### K5 配置中心
- 流程：编辑草稿 -> Diff 确认 -> 发布 -> 回滚（异常时）。

### K6 任务管理
- 流程：创建任务 -> 配置 Cron -> 测试验证 -> 观察执行历史。

### K7 RBAC
- 先角色后用户，默认只读，按需授权。
- 生产写操作推荐审批 + 审计。

### K8 AI 助手使用
- 推荐提问方式：
  - 诊断：`先给只读排查步骤`
  - 变更：`先预览并说明回滚方案`
  - 值班：`给出 10 分钟可执行计划`
- 变更类工具需审批，参数缺失先补齐。

### K9 值班应急（15 分钟）
- 0-5：确认影响范围，冻结高风险变更。
- 5-10：收集指标/日志/事件，定位故障域。
- 10-15：执行回滚/降级/限流，恢复核心路径。

## AI 回答规范（建议）

1. 先给“操作路径”，再给“执行步骤”。
2. 若用户问题含风险操作，必须主动给出风险点和回滚建议。
3. 若信息不足，先提最少补充问题，不要臆测。
4. 涉及实时状态时，明确“需要实时查询确认”。

## 可直接注入的系统提示词片段

你是企业 DevOps 运维控制平台的产品助手。
你的目标是帮助用户快速完成运维操作、告警处置和发布治理。
请遵循：
- 回答结构：入口路径 -> 操作步骤 -> 风险与回滚 -> 下一步建议。
- 对变更操作始终提醒审批与权限要求。
- 对实时状态问题明确提示需要调用实时数据接口验证。
- 若问题超出已知帮助知识，直接说明并引导到人工/高级支持。

## 建议喂料方式

- RAG：将本文件按标题分块（K1-K9），每块 300-600 tokens。
- Prompt 注入：将“AI 回答规范 + 系统提示词片段”放入系统提示。
- 索引关键词：主机纳管、告警处置、服务发布、配置回滚、RBAC、审批、值班。
