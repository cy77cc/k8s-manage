# Proposal: AI助手系统优化

## Summary

对AI助手系统进行全面优化，涵盖四个核心功能：Eino框架废弃方法迁移、实时权限审批、RAG知识库构建、SKILLS支持。旨在提升系统可维护性、审批安全性、AI智能水平和操作便捷性。

## Motivation

### 问题背景

1. **技术债务**: 项目使用了 eino v0.7.36，其中 `Orchestrator` 已被标记为废弃，推荐使用 Graph-based 编排。同时 `react.NewAgent` 的配置方式也有更新，当前代码未跟进。

2. **审批流程缺陷**: 当前审批是同步阻塞式的，遇到需要审批的操作时直接中断流式响应，用户需在前端手动操作后重发请求。审批人选择缺乏基于对象权限的智能匹配，且无超时机制。

3. **AI能力受限**: AI助手缺乏知识检索增强能力，无法利用历史工具调用经验、平台资产信息、故障排查案例等知识来提升响应质量。

4. **缺乏技能抽象**: 当前只有工具和专家两级抽象，缺乏更高级的"技能"概念，无法将常见的多步骤操作流程封装为可复用的技能。

### 业务价值

| 功能 | 业务价值 |
|------|----------|
| Eino迁移 | 消除技术债务，跟进框架最新特性，提升可维护性 |
| 确认-执行审批 | 提升安全性，防止误操作，类似Claude Code的交互体验 |
| RAG知识库 | 提升AI响应准确率，减少重复问题处理时间 |
| SKILLS支持 | 简化常见操作流程，降低用户学习成本 |

## Scope

### In Scope

1. **Eino废弃方法迁移**
   - 移除 `internal/ai/experts/orchestrator.go` 中的废弃 `Orchestrator`
   - 完全迁移到 `internal/ai/graph` 的 Graph Runner
   - 更新 `react.NewAgent` 的配置方式（考虑使用 `WithTools` 选项函数）
   - 评估并处理 v0.8 breaking changes

2. **确认-执行审批机制**
   - 变更操作预览生成（展示工具、参数、影响范围、风险级别）
   - 用户确认机制（无论是否有权限，变更操作都需确认）
   - 风险分级审批（低/中/高风险对应不同审批流程）
   - 基于 Casbin 的对象权限检查（中风险以上操作）
   - 审批请求实时推送（WebSocket）
   - 确认和审批超时机制

3. **RAG知识库**
   - Milvus 向量数据库集成
   - 三类知识 Collection：工具调用示例、平台资产索引、故障排查案例
   - 定时批量更新机制
   - AI对话时的知识检索增强

4. **SKILLS支持**
   - YAML配置文件定义技能
   - 技能路由与参数提取
   - 技能执行器（含审批步骤处理）
   - 与现有 Expert/Tool 系统集成

### Out of Scope

- SKILLS执行历史记录和回放
- 多租户知识库隔离
- 知识库的实时增量更新
- 用户自定义技能（仅管理员配置）

## Risks and Mitigations

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Eino v0.8 breaking changes | 升级后功能异常 | 先在测试环境验证，保留回滚方案 |
| Milvus 部署复杂度 | 运维成本增加 | 使用 Docker Compose 标准化部署，提供监控 |
| WebSocket 连接管理 | 连接断开导致审批丢失 | 实现重连机制，审批状态持久化 |
| SKILLS 配置错误 | 用户操作失败 | YAML schema 验证，配置预检查接口 |

## Success Criteria

1. **Eino迁移**: 所有废弃 API 调用已替换，测试覆盖率达到现有水平
2. **确认-执行审批**: 变更操作确认响应时间 < 5s，风险分级正确执行，超时机制正常工作
3. **RAG知识库**: Milvus 成功集成，知识检索增强生效
4. **SKILLS支持**: 至少配置 3 个常用技能并验证可用
